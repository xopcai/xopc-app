# Product

## Register

product

## Users

XOPC users operate a self-hosted gateway from a mobile device. They manage notes, chat, inbox items, sessions, and gateway connectivity while moving between local network and remote access contexts. The interface needs to support repeated, practical work with minimal distraction.

## Product Purpose

The mobile app is the primary client for interacting with a user-hosted xopc gateway. It should make content capture, review, editing, synchronization, and gateway control feel reliable, fast, and clear on a phone.

## Brand Personality

Calm, precise, trustworthy. The product should feel like a capable native productivity tool: restrained in chrome, clear in state, and confident without decoration.

## Anti-references

Avoid marketing-style hero layouts inside the app, decorative gradients, playful or gamified treatments, neon/cyberpunk styling, hidden gestures without visible affordances, and one-off interaction patterns that diverge from shared mobile primitives.

## Design Principles

- Content leads; navigation and controls support the task without competing for attention.
- One hand matters; primary actions should be reachable and stable around safe areas and keyboard changes.
- State is explicit; sync, saving, selection, errors, disabled controls, and editing mode should be visible without noise.
- Patterns repeat; similar lists, editors, sheets, and action bars should use the same interaction contract.
- Intelligence is embedded; assisted behavior should appear as useful operational state, not decorative performance.

## Accessibility & Inclusion

Target accessible mobile defaults: clear contrast, 44x44 minimum touch targets, readable system typography, reduced-motion respect, stable layouts during keyboard changes, and no hidden critical actions that depend only on gesture discovery.
